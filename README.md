## 游戏玩法

棋盘由 3 × 3 九宫格组成，卡牌默认按外围顺时针、最后中宫的顺序结算：

```text
1   2   3
8   9   4
7   6   5
```

1. 拖拽交换九宫格中的卡牌，规划触发顺序。
2. 触发牌提供分数并进入牌型队列；对子、顺子、同花等牌型会放大当前窗口分。
3. 修饰牌可以添加词条或递归触发其他格子，形成跨格连锁。
4. 每关初始拥有 3 次触发与 3 次弃牌，在触发次数耗尽前达到目标分即可通关。

普通关结束后进入商店，每 3 关出现一次 Boss。完整流程共 24 关，包含 **102 张触发牌、50 张修饰牌、50 张功能牌与 15 种牌型**。

## 技术实现

### 数据流向

```text
XLSX → CSV → SO
```

策划配置表格，通过自动化工具导入，XLSX为权威数据源，CSV为中间产物，SO为最终产物，都不需要手动修改。

### 存档

通过Json进行数据持久，采用阶段存档，方便SL大法。

<!-- Json文件展示 -->

### 随机流

通过才分不同的随机流来达到随机的稳定性，如抽牌，Boss，事件，他们用不同的随机流，互不干扰。

### 触发的实现思路

因为卡牌数量过多，不能为每个卡牌都设计一个类，于是采用id->函数的方式来实现的，就是说开始游戏时通过反射，将id和函数对应，然后当卡牌触发时将卡牌数据灌入触发器，触发器先拿到这个卡牌的id，然后差字典，找到对应的函数，执行触发逻辑。

### 工程技术

- Assembly Definition：拆分 `Core`、`ScriptableObject`、`Component`、`Manager` 与 `App`。
- VContainer + Stratum：通过依赖注入装配运行时服务，调用方只依赖窄接口。
- Addressables：统一配置、Prefab 与面板的资源定位、异步加载和句柄缓存。
- UniTask：串联启动、资源、UI、牌堆与触发流程，并管理取消和异步生命周期。

`Unity 2022.3.60f1c1` · `C#` · `URP 14` · `VContainer 1.17.0` · `Addressables 1.22.3` · `UniTask 2.5.10`
