# Grid Card

> 在九宫格中编排卡牌，按环形顺序触发效果，用扑克牌型构筑得分连锁。

**Grid Card** 是一款单机牌组构筑策略游戏。玩家需要在有限的触发与弃牌次数内完成关卡目标，并通过商店、功能牌、跳关奖励和 Boss 事件持续调整牌组。

## 游戏规则

棋盘为 3 × 3 九宫格，默认触发顺序为：

```text
1   2   3
8   9   4
7   6   5

结算：1 → 2 → 3 → 4 → 5 → 6 → 7 → 8 → 9
```

1. 卡牌进入九宫格后，可以拖拽交换位置，提前规划触发顺序。
2. 每次触发会消耗 1 次触发次数，并按顺序结算棋盘上的卡牌与格子效果。
3. 手牌提供基础分并进入牌型队列；命中对子、顺子、同花等牌型时，倍率作用于当前窗口分。
4. 未达到目标分且仍有触发次数时，棋盘弃牌并重新补满；玩家也可以消耗弃牌次数主动换盘。
5. 在触发次数耗尽前达到目标分即可通关，否则本局失败。每关初始拥有 3 次触发与 3 次弃牌。

普通关通关后进入商店；每 3 关出现一次 Boss。击败 Boss 后可在长期事件与固定金币之间选择奖励。完整流程共 24 关。

## 卡牌与构筑

| 类型 | 作用 |
| --- | --- |
| 触发牌 | 得分、进入或编辑牌型队列，也可以改变跨格结算节奏 |
| 修饰牌 | 添加前置/后置词条，或递归触发九宫格中的其他卡牌 |
| 功能牌 | 独立于战斗牌库，用于换位、改牌、控制牌型、操作商店等即时决策 |

当前内容包含 **102 张触发牌、50 张修饰牌、50 张功能牌与 15 种牌型**。卡牌顺序、牌型队列、递归触发和跨格效果共同构成主要策略空间。

## 技术实现

### 工程架构

- **程序集分层**：使用 Assembly Definition 拆分 `Game.Core`、`Game.ScriptableObject`、`Game.Component`、`Game.Manager` 与 `Game.App`，明确契约、配置、表现、运行时系统和启动入口的职责。
- **依赖注入**：基于 VContainer 与 Stratum，在 `GameLifetimeScope` 中按 `ManagerOrder` 统一注册运行时服务；调用方只依赖窄接口。
- **Addressables**：通过资源键管理配置、Prefab 与面板的异步加载；构建 Player 时自动重建 Addressables Content，成功句柄按应用生命周期缓存。
- **UniTask**：串联启动、面板、资源、牌堆和触发流程；异步操作通过取消、版本号与 owner 校验阻止过期流程写入新状态。

### 核心系统

- **确定性随机**：由 `MasterSeed + 稳定流 Key` 派生牌库、商店、Boss、奖励等独立随机流；存档记录各流消费次数，使同一保存边界可以稳定重放。
- **存档与事务**：JSON 存档与运行时聚合根分离；关键流程同时快照存档、关卡会话、棋盘、牌堆和随机进度，异常或取消时整体回滚。
- **触发引擎**：统一编排九宫格遍历、递归触发、词条队列、牌型倍率、Replay、Boss 规则与执行顺序，完整触发成功后才提交结果。
- **数据驱动**：配置链路为 `XLSX → CSV → DefinitionSO → Runtime`；数值、权重和文案由配置维护，行为通过定义 ID 映射到强类型处理函数。

## 技术栈

`Unity 2022.3.60f1c1` · `C#` · `URP 14` · `VContainer 1.17.0` · `Addressables 1.22.3` · `UniTask 2.5.10`

## 运行游戏

下载仓库中的 Windows 构建版本，解压后运行 `Grid Card.exe`。试玩不需要安装 Unity。
